> 💡 **¿Por qué PromptHub?**
>
> PromptHub no es solo un gestor de prompts, es un **centro de distribución de skills de IA**. Gestiona tus prompts y skills SKILL.md, e instálalos con un clic en Claude Code, Cursor, Windsurf, Codex y 15+ herramientas de codificación IA. Todos los datos se almacenan localmente, priorizando la privacidad.

## ✨ Características

### 📝 Gestión de Prompts

- Crea, edita y elimina con organización por carpetas y etiquetas
- Guardado automático del historial con visualización y recuperación
- Variables de plantilla `{{variable}}` con reemplazo dinámico
- Búsqueda de texto completo, favoritos, adjuntos multimedia

### 🧩 Gestión de Skills 🆕

- **Tienda de Skills**: 20+ skills seleccionados (de Anthropic, OpenAI, etc.)
- **Instalación multiplataforma**: Instalación con un clic en Claude Code, Cursor, Windsurf, Codex, Kiro, Gemini CLI, Qoder, QoderWork, CodeBuddy y 15+ plataformas
- **Escaneo local**: Detección automática de SKILL.md locales, vista previa e importación
- **Modo Symlink/Copia**: Enlace simbólico o copia independiente
- **Traducción IA**: Traducción inmersiva/completa del contenido de skills
- **Filtrado por etiquetas**: Filtrado rápido de skills por etiquetas

### 🤖 Capacidades de IA

- Pruebas multimodelo integradas con diversos proveedores
- Pruebas comparativas en paralelo con múltiples modelos
- Generación y refinamiento de contenido de skills con IA

### 💾 Datos y Sincronización

- Todos los datos almacenados localmente, privacidad protegida
- Copia de seguridad completa/restauración (`.phub.gz`)
- Sincronización en la nube WebDAV
- Oscuro/Claro/Sistema, 7 idiomas, multiplataforma

### Primer inicio en macOS

Dado que la aplicación no está notarizada por Apple, es posible que al abrirla por primera vez veas el mensaje **"PromptHub está dañado y no se puede abrir"** o **"No se puede verificar el desarrollador"**.

**Solución (recomendada)**: Abre la Terminal y ejecuta el siguiente comando para omitir la verificación de Gatekeeper:

```bash
sudo xattr -rd com.apple.quarantine /Applications/PromptHub.app
```

> 💡 **Consejo**: Si la aplicación está instalada en otra ubicación, sustituye la ruta por la ruta de instalación real.

**O bien**: Ve a "Ajustes del sistema" → "Privacidad y seguridad" → desplázate hasta la sección de Seguridad → haz clic en "Abrir de todos modos".

## CLI de línea de comandos

PromptHub ahora ofrece tanto GUI como CLI.

> ⚠️ **Comportamiento actual**
>
> - Después de instalar la app de escritorio y abrir PromptHub una vez, la aplicación instala automáticamente el comando `prompthub`
> - Tras volver a abrir la terminal, ya puedes usar `prompthub --args` directamente
> - La ejecución desde el código fuente y el bundle CLI compilado siguen disponibles para desarrollo y depuración

### Usarlo directamente después de instalar la app de escritorio

```bash
prompthub --help
prompthub prompt list
prompthub skill list
prompthub --output table prompt search SEO --favorite
```

> 💡 **Consejo**
>
> - Si acabas de instalar la app de escritorio, abre PromptHub una vez primero
> - Si tu terminal actual todavía no encuentra `prompthub`, ciérrala y ábrela de nuevo

### Ejecutar desde el código fuente

```bash
pnpm install
pnpm cli:dev -- --help
pnpm cli:dev -- prompt list
pnpm cli:dev -- skill list
pnpm cli:dev -- skill install ~/.claude/skills/my-skill
```

### Usar el bundle CLI compilado

```bash
pnpm build
node out/cli/prompthub.cjs --help
node out/cli/prompthub.cjs prompt list
node out/cli/prompthub.cjs skill list
```

### Opciones comunes

- `--output json|table`
- `--data-dir /path/to/user-data`
- `--app-data-dir /path/to/app-data`

### Comandos compatibles

- `prompt list|get|create|update|delete|search`
- `skill list|get|install|scan|delete|remove`

### Notas

- La CLI lee y escribe directamente la base de datos local de PromptHub y el repositorio de skills gestionado
- Es útil para scripts, operaciones por lotes, importaciones, escaneos y automatización
- La app de escritorio instala el wrapper del comando shell en el primer arranque
- Si más adelante mueves la aplicación, volver a abrir PromptHub actualizará la ruta del wrapper

## 🚀 Inicio rápido

### 1. Crear un Prompt

Haz clic en el botón "Nuevo" y completa:

- **Título** - Nombre del prompt
- **Descripción** - Breve explicación del uso
- **System Prompt** - Establece el rol de la IA (opcional)
- **User Prompt** - El contenido real del prompt
- **Etiquetas** - Para categorización y búsqueda

### 2. Usar variables

Usa la sintaxis `{{nombre_variable}}` en tus prompts:

```
Por favor, traduce el siguiente texto de {{source_lang}} a {{target_lang}}:

{{text}}
```

### 3. Copiar y usar

Selecciona un prompt y haz clic en "Copiar" para llevar el contenido al portapapeles.

### 4. Gestión de versiones

El historial de edición se guarda automáticamente. Haz clic en "Historial" para ver y restaurar versiones anteriores.
