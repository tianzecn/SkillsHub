> 💡 **Pourquoi PromptHub ?**
>
> PromptHub n'est pas qu'un gestionnaire de prompts, c'est un **centre de distribution de skills IA**. Gérez vos prompts et skills SKILL.md, et installez-les en un clic sur Claude Code, Cursor, Windsurf, Codex et 15+ outils de codage IA. Toutes les données sont stockées localement, vie privée en priorité.

## ✨ Fonctionnalités

### 📝 Gestion des Prompts

- Créer, modifier, supprimer avec organisation par dossiers et tags
- Sauvegarde automatique de l'historique avec visualisation et restauration
- Variables de modèle `{{variable}}` avec remplacement dynamique
- Recherche plein texte, favoris, pièces jointes multimédia

### 🧩 Gestion des Skills 🆕

- **Boutique de Skills** : 20+ skills sélectionnés (d'Anthropic, OpenAI, etc.)
- **Installation multiplateforme** : Installation en un clic sur Claude Code, Cursor, Windsurf, Codex, Kiro, Gemini CLI, Qoder, QoderWork, CodeBuddy et 15+ plateformes
- **Scan local** : Détection automatique des SKILL.md locaux, aperçu et importation
- **Mode Symlink/Copie** : Lien symbolique ou copie indépendante
- **Traduction IA** : Traduction immersive/complète du contenu des skills
- **Filtrage par tags** : Filtrage rapide des skills par tags

### 🤖 Capacités IA

- Tests multi-modèles intégrés avec divers fournisseurs
- Tests comparatifs en parallèle avec plusieurs modèles
- Génération et amélioration de contenu de skills par IA

### 💾 Données et Synchronisation

- Toutes les données stockées localement, vie privée protégée
- Sauvegarde/restauration complète (`.phub.gz`)
- Synchronisation cloud WebDAV
- Sombre/Clair/Système, 7 langues, multiplateforme

### Premier démarrage sur macOS

Comme l'application n'est pas notariée par Apple, il est possible qu'au premier démarrage vous voyiez le message **"PromptHub est endommagé et ne peut pas être ouvert"** ou **"Impossible de vérifier le développeur"**.

**Solution (recommandée)** : Ouvrez le Terminal et exécutez la commande suivante pour contourner la vérification de Gatekeeper :

```bash
sudo xattr -rd com.apple.quarantine /Applications/PromptHub.app
```

> 💡 **Conseil** : Si l'application est installée ailleurs, remplacez le chemin par le chemin d'installation réel.

**Ou bien** : Allez dans "Réglages Système" → "Confidentialité et sécurité" → faites défiler jusqu'à la section Sécurité → cliquez sur "Ouvrir quand même".

## CLI en ligne de commande

PromptHub fournit maintenant à la fois une GUI et une CLI.

> ⚠️ **Comportement actuel**
>
> - Après avoir installé l'application de bureau et lancé PromptHub une première fois, l'application installe automatiquement la commande `prompthub`
> - Après avoir rouvert votre terminal, vous pouvez utiliser directement `prompthub --args`
> - L'exécution depuis les sources et le bundle CLI compilé reste disponible pour le développement et le débogage

### Utilisation directe après installation de l'application de bureau

```bash
prompthub --help
prompthub prompt list
prompthub skill list
prompthub --output table prompt search SEO --favorite
```

> 💡 **Conseil**
>
> - Si vous venez juste d'installer l'application, lancez PromptHub une première fois
> - Si votre terminal actuel ne trouve pas encore `prompthub`, fermez-le puis ouvrez-en un nouveau

### Exécuter depuis les sources

```bash
pnpm install
pnpm cli:dev -- --help
pnpm cli:dev -- prompt list
pnpm cli:dev -- skill list
pnpm cli:dev -- skill install ~/.claude/skills/my-skill
```

### Utiliser le bundle CLI compilé

```bash
pnpm build
node out/cli/prompthub.cjs --help
node out/cli/prompthub.cjs prompt list
node out/cli/prompthub.cjs skill list
```

### Options courantes

- `--output json|table`
- `--data-dir /path/to/user-data`
- `--app-data-dir /path/to/app-data`

### Commandes prises en charge

- `prompt list|get|create|update|delete|search`
- `skill list|get|install|scan|delete|remove`

### Notes

- La CLI lit et écrit directement dans la base de données locale de PromptHub et dans le dépôt de skills géré
- Elle convient aux scripts, aux opérations par lot, aux imports, aux scans et à l'automatisation
- L'application de bureau installe automatiquement le wrapper de commande shell au premier lancement
- Si vous déplacez ensuite l'application, relancer PromptHub mettra à jour la cible du wrapper

## 🚀 Démarrage rapide

### 1. Créer un Prompt

Cliquez sur le bouton "Nouveau" et remplissez :

- **Titre** - Nom du prompt
- **Description** - Brève explication de l'usage
- **System Prompt** - Définit le rôle de l'IA (optionnel)
- **User Prompt** - Le contenu réel du prompt
- **Tags** - Pour la catégorisation et la recherche

### 2. Utiliser des variables

Utilisez la syntaxe `{{nom_variable}}` dans vos prompts :

```
Veuillez traduire le texte suivant de {{source_lang}} en {{target_lang}} :

{{text}}
```

### 3. Copier et utiliser

Sélectionnez un prompt et cliquez sur "Copier" pour mettre le contenu dans le presse-papiers.

### 4. Gestion des versions

L'historique d'édition est sauvegardé automatiquement. Cliquez sur "Historique" pour voir et restaurer les versions précédentes.

- [x] **Détection des mises à jour des Skills du store** : les Skills installés depuis le store conservent un hash du contenu installé et peuvent vérifier le `SKILL.md` distant
